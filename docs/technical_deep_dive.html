Neuro Exam App • Architecture

1. Executive Summary

The Neuro Exam App is a high-performance, Modern Web Application (MWA) designed for seamless assessment delivery. Built on a robust React 19 foundation, it leverages a custom-built Parsing Engine to interpret raw text into interactive exam sessions instantly. The system prioritizes User Experience (UX) through fluid animations, instant feedback loops, and a "Premium" glassmorphism-inspired design system.

Key Objective: To provide an "Ultimate" look and feel while adhering to strict production standards (CI/CD, Docker, Type Safety).

3. The Parsing Engine

At the heart of the application lies the ParsingEngine class (located in src/shared/lib/utils.ts). Unlike traditional apps that require structured JSON uploads, Neuro Exam intelligently deciphers unstructured plain text.

3.1 Algorithm Breakdown

The parser utilizes a multi-pass regex strategy to identify question blocks.

graph TD A[Raw Text Input] --> B{Is JSON?} B -->|Yes| C[Direct Import] B -->|No| D[Text Normalization] D --> E[Regex Split Header] E --> F[Block Iterator] F --> G{Parse Line Mode} G -->|Header| H[Extract QID] G -->|Body| I[Accumulate Text] G -->|Footer| J[Detect Answer/Options] J --> K[Validation Check] K --> L[Final Question Object]

3.2 Regex Implementation

The engine uses Lookahead Assertions to split questions without consuming the delimiters, ensuring no data loss.

// Regex to identify new question blocks const splitPattern = /\n(?=Q\d+[\.:\)\s])/i; // Regex to capture "Correct Answer" tags const answerPattern = /^Correct Answer:\s*([A-D])/i; // Fallback for bracketed format const bracketPattern = /\[(?:Correct )?Answer:\s*([A-D])\]/i;
Resilient Parsing: The logic adapts to missing "Options" lines by detecting embedded `(A) / (B)` patterns, automatically converting Error Detection questions into a clickable format.

4. Result Analysis System

The ResultAnalysis.tsx module transforms raw exam data into actionable insights effectively gamifying the assessment process.

4.1 Scoring Logic

Scores are calculated in real-time using functional array methods for O(n) efficiency. The system tracks:

  • Accuracy: Correct vs. Incorrect vs. Skipped.
  • Pass/Fail Metric: Dynamic thresholding (default 40%).
  • Gamification: Confetti triggers via canvas-confetti for passing scores.

4.2 Code Structure (Snippet)

const correctCount = questions.filter( q => answers[q.id] === q.correctAnswer ).length; const score = Math.round((correctCount / total) * 100); // Effect hook for confetti celebration useEffect(() => { if (score >= 40) triggerConfetti(); }, [score]);

4.3 PDF Generation Integration

Users can export these results using the client-side PDFGenerator class. This ensures data privacy as no data is sent to a server for processing; the PDF is compiled directly in the browser memory using jspdf and jspdf-autotable.

5. Conclusion

The Neuro Exam App represents a shift towards "Smart Client" architectures. By moving heavy parsing and generation logic to the browser, we achieve near-zero latency and infinite scalability. The premium UI wrapper ensures that this power is delivered with elegance.
